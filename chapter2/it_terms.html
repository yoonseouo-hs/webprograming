IT 용어 50선

IT 분야에서 자주 사용되는 용어 50개 이상의 설명을 모았습니다.

IT 용어 목록

Algorithm (알고리즘)
문제를 해결하기 위한 절차나 방법의 집합.
API (Application Programming Interface)
응용 프로그램에서 사용할 수 있도록 운영체제나 프로그래밍 언어가 제공하는 기능을 제어하는 인터페이스.
Array (배열)
같은 타입의 데이터가 연속적으로 저장된 자료구조.
Bandwidth (대역폭)
네트워크에서 단위 시간당 전송할 수 있는 데이터의 양.
Big Data (빅데이터)
기존 데이터베이스 관리 도구로 처리하기 어려운 대용량 데이터.
Binary (이진수)
0과 1로만 구성된 수 체계.
Bit
컴퓨터가 처리하는 정보의 최소 단위.
Blockchain (블록체인)
분산형 데이터 저장 기술로, 거래 내역을 블록 단위로 연결하여 관리.
Boolean (불린)
참(True)과 거짓(False) 두 가지 값만 가지는 논리 자료형.
Browser (브라우저)
웹페이지를 탐색하고 표시하는 소프트웨어.
Bug (버그)
프로그램 내의 오류나 결함.
Cache (캐시)
데이터를 임시로 저장하여 빠르게 접근할 수 있게 하는 메모리.
Cloud Computing (클라우드 컴퓨팅)
인터넷을 통해 서버, 스토리지, 데이터베이스 등 IT 자원을 제공하는 서비스.
Compiler (컴파일러)
프로그래밍 언어로 작성된 소스 코드를 기계어로 변환하는 프로그램.
Cookie (쿠키)
웹사이트가 사용자의 브라우저에 저장하는 작은 데이터 조각.
CPU (Central Processing Unit)
컴퓨터의 중앙 처리 장치로, 연산과 제어를 담당.
CSS (Cascading Style Sheets)
웹페이지의 스타일과 레이아웃을 지정하는 언어.
Database (데이터베이스)
구조화된 데이터의 집합.
Debugging (디버깅)
프로그램의 오류를 찾아 수정하는 과정.
DNS (Domain Name System)
도메인 이름을 IP 주소로 변환해주는 시스템.
DOM (Document Object Model)
HTML, XML 문서를 객체로 표현한 구조.
Encryption (암호화)
데이터를 안전하게 보호하기 위해 내용을 변환하는 기술.
Ethernet (이더넷)
근거리 통신망(LAN)에서 사용되는 유선 통신 규격.
Firewall (방화벽)
네트워크를 보호하기 위한 보안 시스템.
Framework (프레임워크)
소프트웨어 개발을 위한 기본 구조와 규칙을 제공하는 틀.
FTP (File Transfer Protocol)
파일을 전송하기 위한 통신 규약.
Function (함수)
특정 작업을 수행하는 코드의 집합.
Git
분산 버전 관리 시스템.
GUI (Graphical User Interface)
그래픽을 이용해 사용자와 소통하는 인터페이스.
HTML (HyperText Markup Language)
웹페이지의 구조를 정의하는 마크업 언어.
HTTP (HyperText Transfer Protocol)
웹에서 데이터를 주고받기 위한 프로토콜.
HTTPS (HyperText Transfer Protocol Secure)
보안이 강화된 HTTP 프로토콜.
IP Address (IP 주소)
네트워크에서 장치를 식별하는 고유한 주소.
Java
객체지향 프로그래밍 언어 중 하나.
JavaScript
웹페이지에 동적인 기능을 추가하는 스크립트 언어.
JSON (JavaScript Object Notation)
데이터를 저장하고 전송하기 위한 경량의 데이터 형식.
Kernel (커널)
운영체제의 핵심 부분으로, 하드웨어와 소프트웨어를 연결.
LAN (Local Area Network)
좁은 지역 내에서 컴퓨터를 연결하는 네트워크.
Library (라이브러리)
특정 기능을 수행하는 코드의 집합.
Linux
오픈 소스 운영체제의 한 종류.
Loop (반복문)
동일한 작업을 반복 수행하는 명령문.
Machine Learning (머신러닝)
컴퓨터가 데이터를 통해 스스로 학습하는 기술.
Network (네트워크)
여러 컴퓨터가 서로 연결되어 정보를 주고받는 구조.
Object (객체)
데이터와 그 데이터를 처리하는 함수를 함께 묶은 단위.
Open Source (오픈소스)
소스 코드가 공개되어 누구나 사용할 수 있는 소프트웨어.
Operating System (운영체제)
컴퓨터 하드웨어와 소프트웨어를 관리하는 시스템 소프트웨어.
Packet (패킷)
네트워크를 통해 전송되는 데이터의 단위.
Phishing (피싱)
개인 정보를 빼내기 위한 사기 수법.
Pixel (픽셀)
디지털 이미지를 구성하는 최소 단위.
Protocol (프로토콜)
네트워크에서 데이터를 주고받기 위한 규칙.
Query (쿼리)
데이터베이스에서 정보를 요청하는 명령.
RAM (Random Access Memory)
임의 접근이 가능한 주기억장치.
Repository (저장소)
소스 코드나 데이터를 저장하는 공간.
Router (라우터)
네트워크에서 데이터의 경로를 결정하는 장치.
Runtime (런타임)
프로그램이 실행되는 시간 또는 환경.
Script (스크립트)
특정 작업을 자동화하기 위한 간단한 프로그램.
SDK (Software Development Kit)
소프트웨어 개발에 필요한 도구 모음.
Server (서버)
네트워크를 통해 서비스를 제공하는 컴퓨터.
Source Code (소스 코드)
사람이 읽을 수 있는 프로그래밍 코드.
SQL (Structured Query Language)
데이터베이스를 관리하고 조작하는 언어.
TCP/IP
인터넷에서 표준으로 사용되는 통신 프로토콜.
UI (User Interface)
사용자와 시스템이 상호작용하는 방식.
URL (Uniform Resource Locator)
웹에서 자원의 위치를 나타내는 주소.
UX (User Experience)
사용자가 제품이나 서비스를 이용하면서 느끼는 경험.
Variable (변수)
값을 저장할 수 있는 메모리 공간.
Virtual Machine (가상 머신)
물리적 컴퓨터와 독립적으로 동작하는 소프트웨어 기반 컴퓨터.
Virus (바이러스)
컴퓨터에 피해를 주는 악성 소프트웨어.
Web Server (웹 서버)
웹페이지를 제공하는 서버.
Wi-Fi
무선으로 네트워크에 접속할 수 있는 기술.
XML (eXtensible Markup Language)
데이터를 저장하고 전송하기 위한 마크업 언어.
